# MOTION LAB 運用手順書

このサイトを公開後に「記事リンクを追加する」「文言を直す」ための手順書です。
プログラミングの知識がなくても、**決まった場所の文字を書き換えるだけ**で更新できます。

---

## 0. まず覚える「更新の全体の流れ」

このサイトは次の3ステップで更新されます。

```
① ファイルの文字を書き換える（下記の各手順）
        ↓
② GitHub に保存（push）する
        ↓
③ Vercel が自動でサイトを作り直す（30秒〜1分で本番に反映）
```

- **公開URL**: https://motion-lab-kappa.vercel.app/
- **①の編集**はパソコン上のファイルを直すか、GitHubの画面上でも直せます（後述）。
- **②のpush**は「GitHub Desktop」アプリを使うのが一番かんたんです。
- **③は自動**なので、あなたは何もしなくてOKです。

> 💡 GitHubの画面上で直接編集すれば、①と②が同時に終わります（パソコンにアプリ不要）。
> 慣れないうちは **「GitHubのWeb画面で直接編集」** が一番安全でおすすめです（手順は §5 参照）。

## 1. ブログ記事リンクの追加・修正

3D骨格の各部位をタップすると出てくる「READ ARTICLE（関連記事）」のリンクを管理します。

### 📄 編集する場所
```
src/data/regions.ts
```

### 中身の見方
部位ごとに、次のような「かたまり」が10個並んでいます（首・肩・肘…）。

```ts
  {
    id: 'neck',
    label: '首・頸椎',
    en: 'NECK',
    position: [0, 2.42, 0.22],
    summary:
      '頭の重さ約5kgを支える司令塔。スマホ姿勢による頸椎の前方化は…',
    signs: ['朝起きると首が重い', 'デスクワーク後の頭痛', '振り向くと詰まり感がある'],
    articleTitle: 'ストレートネックは「首」だけの問題ではない',
    articleUrl: 'https://example.com/insights/neck',
  },
```

### ✏️ よくある編集

| やりたいこと | 直す行 | 例 |
|---|---|---|
| 記事リンク先を本物にする | `articleUrl:` | `'https://あなたのブログ.com/neck'` |
| リンクに表示するタイトルを変える | `articleTitle:` | `'首こりの本当の原因とは'` |
| パネルの説明文を変える | `summary:` | 好きな文章に |
| 「こんなサイン」の項目を変える | `signs:` | `['朝、首が重い', '頭痛が続く']` |
| 部位の日本語名を変える | `label:` | `'首まわり'` |

### ⚠️ 触ってはいけない行
- `id:` … システム上の名前。変えると壊れます。
- `position:` … 3Dモデル上の光る点の座標。変えると点の位置がズレます。
- `en:` … 変えても表示は変わりますが、通常は不要。

### 🔑 書き換えの鉄則（これだけ守れば安全）
1. **文字は必ず `'シングルクォート'` で囲む**（`'…'` の中だけを書き換える）。
2. **行末の `,`（カンマ）は消さない**。
3. 文章の中で `'` を使いたいときは `「」` を使う（例: `'首の「こり」について'`）。
4. `signs:` は `['項目1', '項目2', '項目3']` の形。項目はカンマ区切りで増減できます。

> 部位を丸ごと増やす／減らすのは表示バランスに影響するため、必要なときは開発者に相談してください（現在は10部位固定でデザイン調整済み）。

## 2. 各ページの文言（見出し・説明文）の修正

セクションごとに担当ファイルが分かれています。直したい文字を探して書き換えます。

| 直したい場所 | ファイル |
|---|---|
| トップの大見出し「カラダは、遊べる実験室。」/ キャッチコピー / ボタン文字 | `src/sections/Hero.tsx` |
| ヘッダーのメニュー名 / 一番下のフッター文（免責文など） | `src/App.tsx` |
| 01 BODY MAP の見出し・説明文 | `src/sections/BodyMap.tsx` |
| 02 IAP LAB（腹圧）の見出し・説明・判定コメント | `src/sections/IapLab.tsx` |
| 03 NEURO LAB（神経）の見出し・テスト説明・注意書き | `src/sections/NeuroLab.tsx` |
| 04 FASCIA LAB（筋膜）の見出し・説明 | `src/sections/FasciaLab.tsx` |
| 05 NUTRI LAB（栄養）の見出し・説明・目安の数値 | `src/sections/NutriLab.tsx` |

### ✏️ 直し方
ファイルを開き、画面に出ている**日本語の文字**を探して、その部分だけ書き換えます。
例（Hero.tsx の一部）:
```tsx
<p className="hero-lead">
  読むだけの健康情報は、もう終わり。3Dの骨格に触れ…  ← ここの文章を書き換える
</p>
```

### ⚠️ 注意
- `<p ...>` `</p>` や `className="..."` のような**記号・英語部分は触らない**。挟まれた日本語だけ直す。
- `{ }` `< >` `" "` などの記号を消すと表示が崩れます。迷ったら**その行はいじらない**。
- **医療免責の文言**（「医学的な診断・治療に代わるものではありません」等）は、意味を弱めないよう注意。安易に削除しないでください。

## 3. 変更を本番に反映する（3つの方法）

### 方法A：GitHubのWeb画面で直接編集（初心者に一番おすすめ）
アプリ不要・パソコンの設定不要。§5 に詳しい手順。

### 方法B：GitHub Desktop アプリ
1. パソコンでファイルを編集して保存
2. GitHub Desktop を開くと変更が一覧表示される
3. 左下に短いメモ（例: `記事リンク更新`）を書いて **Commit**
4. 右上の **Push origin** をクリック → 1分ほどで本番反映

### 方法C：ターミナル（慣れている人向け）
```bash
cd /Users/etsuki/Desktop/アプリ開発/motion-lab
git add .
git commit -m "文言を修正"
git push
```

どの方法でも、pushの後 **Vercelが自動で再公開**します。反映確認は公開URLを再読み込み（スーパーリロード: Cmd+Shift+R）してください。

## 4. 事前チェック（任意・パソコンで編集した場合）

文字を書き換えた後、記号を消してしまっていないか自分で確認できます。
```bash
cd /Users/etsuki/Desktop/アプリ開発/motion-lab
npm run build
```
- `✓ built` と出れば **OK（安全に反映できます）**。
- `error` が出たら、直前に書き換えた箇所の**カンマ・クォート・記号**を見直してください。
  （GitHub Web画面編集の場合は、この確認は飛ばしてOK。おかしければVercelがDeploy失敗として教えてくれます）

## 5. 【一番かんたん】GitHubのWeb画面だけで更新する手順

パソコンにアプリを入れず、ブラウザだけで①編集〜②保存まで完了します。

1. https://github.com/tomynote2025/motion-lab を開く
2. フォルダをたどって直したいファイルを開く
   （例: 記事リンクなら `src` → `data` → `regions.ts`）
3. 右上の **鉛筆マーク（✏️ Edit this file）** をクリック
4. 画面上で文字を書き換える
5. 右上の緑ボタン **Commit changes...** をクリック
6. メモ欄はそのままでOK → **Commit changes** を押す
7. 完了。1分ほどで公開URLに反映されます

> この方法なら、間違えても履歴から元に戻せます（GitHubが全部記録しています）。

---

## 6. 困ったときのメモ

- **反映されない** → ブラウザのスーパーリロード（Cmd+Shift+R）。それでもダメなら Vercel の画面でDeployが成功しているか確認。
- **Deployが赤く失敗した** → 直前の編集で記号（カンマ・クォート・かっこ）を壊した可能性大。GitHubの履歴から一つ前に戻すか、開発者に該当ファイル名を伝える。
- **記事リンクをまとめて差し替えたい** → `src/data/regions.ts` の `articleUrl` を10個順に直すだけ。
- **判断に迷う編集**（部位の増減・レイアウト・3D・数値ロジックなど）→ 無理せず開発者に依頼。
